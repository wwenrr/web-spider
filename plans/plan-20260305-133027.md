# Goal
Thiết kế và triển khai đầy đủ tính năng quản lý CDP connection gồm:
- 1 migration tạo bảng lưu thông tin kết nối CDP (ít nhất có `host`, `port`, và các trường cần thiết mở rộng).
- 1 màn hình UI trong app để CRUD bản ghi CDP connection (create, list, update, delete) theo phong cách sidebar hiện tại.

# Assumptions
- Stack giữ nguyên: Peewee + peewee-migrate cho DB; NiceGUI cho UI.
- Màn hình mới sẽ đi theo pattern hiện có của `todo` (domain facade/service/repository + UI page render trong app root với query `?view=`).
- Trường dữ liệu ban đầu của CDP connection (có thể điều chỉnh khi implement):
  - `id` (AutoField)
  - `name` (CharField, tên hiển thị)
  - `host` (CharField, bắt buộc)
  - `port` (IntegerField, bắt buộc)
  - `browser` (CharField, ví dụ `chrome`, mặc định `chrome`)
  - `ws_path` (CharField, mặc định `/devtools/browser` hoặc rỗng)
  - `username` (CharField nullable)
  - `password` (CharField nullable)
  - `is_active` (BooleanField, mặc định `True`)
  - `notes` (TextField nullable)
  - `created_at`, `updated_at` (UTC timestamp)
- Validation cơ bản: host không rỗng, port trong khoảng hợp lệ 1-65535, name không rỗng.
- UI phải tuân thủ `.agents/ui-rule.md` và `.agents/ui-modern-minimalism-sidebar.md`.

# Steps
1. Khảo sát và chốt schema
- So sánh với model `Todo` hiện tại để giữ naming convention/constants nhất quán.
- Chốt danh sách cột, giá trị mặc định, nullable, và index/unique (ví dụ unique theo `name` nếu cần).

2. Tạo migration
- Thêm file migration mới, ví dụ `db/migrations/002_create_cdp_connection.py`.
- Trong `migrate()`: tạo table bằng model mới với `database.create_tables([...], safe=True)`.
- Trong `rollback()`: drop table tương ứng với `safe=True`.

3. Bổ sung model + constants
- Thêm table name constant trong `src/models/constants/table_names.py`.
- Tạo model mới `src/models/cdp_connection.py` kế thừa `BaseModel`, có helper `utc_now()` và đầy đủ field.

4. Bổ sung tầng repository/domain cho CRUD
- Tạo repository mới tại `src/infrastructure/repositories/cdp_connection.py` với hàm:
  - `list_connections()`
  - `create_connection(...)`
  - `update_connection(connection_id, ...)`
  - `delete_connection(connection_id)`
  - (tuỳ chọn) `get_connection(connection_id)`
- Tạo interface/domain service/facade tương ứng theo pattern `todos` để UI gọi qua domain, không gọi trực tiếp repository.
- Thêm validate + error message rõ ràng (không để dữ liệu invalid vào DB).

5. Tích hợp vào app lifecycle
- Wire service tại `src/web_app.py` tương tự `configure_todo_manager(...)`.
- Đảm bảo singleton getter cho repository/service nhất quán cách codebase đang làm.

6. Tạo UI CRUD screen
- Tạo page render mới (ví dụ `src/ui/pages/cdp_connections.py`) với:
  - Form tạo mới (name, host, port, các trường mở rộng)
  - Danh sách record hiện có
  - Inline edit hoặc form edit riêng
  - Nút delete có xác nhận
- Dùng `@ui.refreshable` để refresh list sau thao tác CRUD, pattern giống `todos`.
- Hiển thị thông báo thành công/lỗi qua `show_success` / `show_error`.

7. Gắn route + sidebar navigation
- Thêm route constant mới (ví dụ `ROUTE_CDP_CONNECTIONS = "/cdp-connections"`).
- Cập nhật `src/ui/pages/app.py`:
  - thêm `ViewConfig` cho `cdp_connections`
  - parse và render đúng section mới
  - thêm alias route redirect `/?view=cdp_connections`
- Cập nhật `src/ui/layout/global_layout.py` để có menu item mới trong sidebar, active state đúng.

8. CSS và responsive cho màn hình mới
- Bổ sung class vào `src/ui/static/custom.css` theo 8px spacing system, palette xanh/xám/trắng.
- Đảm bảo desktop + mobile không vỡ layout, giữ mức tương phản và focus state rõ.

9. Hoàn thiện và dọn kiến trúc
- Kiểm tra tuân thủ rule “helper function ở dưới main function gọi nó”.
- Chuẩn hoá import order, typing, naming.

# Validation
- Chạy lint toàn dự án:
  - `poetry run lint`
- Kiểm tra migration:
  - chạy migrate lên DB local và xác nhận table mới được tạo.
  - rollback thử để chắc chắn migration reversible.
- Kiểm tra chức năng UI thủ công:
  - Create 1 connection hợp lệ.
  - Update host/port và các trường mở rộng.
  - Delete connection.
  - Test validation lỗi (host rỗng, port ngoài khoảng).
- Kiểm tra điều hướng:
  - Sidebar hiển thị mục mới, active state đúng.
  - Route alias hoạt động (`/cdp-connections` -> `/?view=cdp_connections`).
