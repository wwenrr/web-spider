# Goal
Implement a Spy products crawl workflow that allows users to submit a product page URL, persists a pending product record, and processes it asynchronously via background job to mark crawl completion.

# Assumptions
- Existing pybgworker setup is the standard async execution path for background jobs in this project.
- For now, crawl execution is mocked: the worker sleeps for 10 seconds, then updates status to `done`.
- `images_url` and `category` are stored as JSON strings (stringified arrays) in SQLite text columns.
- New UI for Spy must use native DOM composition (`ui.element`, `ui.html`, `ui.run_javascript`) instead of NiceGUI Quasar convenience components.

# Steps
1. Add data model and migration:
   - Create `Product` Peewee model with fields: `thumbnail`, `name`, `category`, `price`, `images_url`, `barcode`, `code`, `remote_id`, `remote_url`, `description`, `crawl_status`, `crawl_at`, timestamps.
   - Create migration `003_create_product.py`.
   - Update model constants and exports where needed.
2. Add repository and core layer:
   - Create product repository for create/list/update crawl status/update crawl result.
   - Add `core.products` manager + facade runtime/api similar to existing module patterns.
3. Add background task integration:
   - Add task constant and registry mapping for product crawl job.
   - Implement worker job that sleeps 10 seconds then updates product status to `done` and `crawl_at`.
   - Expose task in `tasks.py`.
4. Wire runtime dependency:
   - Configure product manager in `web_app.py` with repository and default queue.
5. Build Spy products UI:
   - Add a new Spy tab for products crawl.
   - Render native input + button to submit URL.
   - On submit, create pending product record and enqueue crawl job.
   - Render product table/list with crawl status and relevant fields.
6. Validation:
   - Run `poetry run lint`.
   - Ensure no stale imports and all added modules are wired.

# Validation
- `poetry run lint` exits with code 0.
- New product migration file exists and model imports resolve.
- Spy UI shows products crawl tab with URL submission and visible pending/done status transitions after background worker execution.
