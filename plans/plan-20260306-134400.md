# Goal
Thiết kế một cơ chế `managed browser fallback` để thay thế việc mỗi task tự `launch()` local Playwright browser khi không có `CDP Connection`. Mục tiêu là giảm số lượng browser process bị tạo trùng, tránh tràn RAM, và bổ sung một màn hình trong nhóm `Operations` để người dùng quản lý các browser local fallback giống như một tài nguyên vận hành có thể tái sử dụng.

# Assumptions
- Worker hiện tại dùng `pybgworker`, và mỗi task được chạy trong process riêng. Vì vậy cache browser ở mức module/process như trong [playwright_runtime.py](/home/mk/Documents/Github/Dogo-Corp/web-spider/src/infrastructure/adapters/spiders/helpers/playwright_runtime.py) không giúp tái sử dụng local browser giữa các task khác nhau.
- CDP attach vẫn là ưu tiên số 1 khi có connection active. Bài toán cần giải chỉ là nhánh fallback khi không có CDP dùng được.
- Local fallback browser nên được quản lý như một tài nguyên sống lâu hơn vòng đời từng task, có metadata trong DB và có thể nhìn/điều khiển từ UI.
- UI quản lý browser nên đặt trong nhóm `Operations`, cạnh `CDP Connections`, thay vì trộn vào trang Spy hay Products Crawl.
- Không nên để task worker tự giữ browser sống trong process của chính nó, vì worker hiện tại spawn process riêng cho từng task.
- Hướng phù hợp hơn là dùng một browser local chạy tách biệt, expose qua CDP endpoint nội bộ, sau đó crawler attach lại bằng `connect_over_cdp()` như với remote browser.

# Steps
1. Thiết kế lại khái niệm fallback browser.
   - Thay vì `local launch per task`, chuyển sang `managed local browser` chạy nền và expose CDP endpoint riêng.
   - Runtime chọn nguồn browser theo thứ tự:
     - `CDP Connection` active do user cấu hình
     - `Managed Browser` local đang running và active
     - nếu chưa có managed browser running thì khởi động nó, lưu trạng thái, rồi attach lại qua CDP
   - Như vậy mọi nhánh đều thống nhất về một cơ chế attach browser qua CDP, tránh phân nhánh phức tạp trong spider.

2. Thêm bảng DB mới để quản lý browser local fallback.
   - Tạo migration mới và model mới, ví dụ `managed_browser`.
   - Trường nên có:
     - `id`
     - `name`
     - `browser_type` ví dụ `chromium`
     - `host` mặc định `127.0.0.1`
     - `port`
     - `user_data_dir`
     - `headless`
     - `launch_args` nullable text/json
     - `is_active`
     - `status` enum: `stopped`, `starting`, `running`, `failed`
     - `pid` nullable int
     - `last_started_at`
     - `last_seen_at`
     - `notes` nullable text
     - `created_at`, `updated_at`
   - Nên có unique index cho `port` và có thể cho `name`.

3. Tạo service/repository/facade cho `managed_browser`.
   - Service cần hỗ trợ:
     - list browsers
     - create browser config
     - update browser config
     - delete browser config khi đang `stopped`
     - start browser
     - stop browser
     - mark health/status
   - Cần tách rõ:
     - DB lưu cấu hình + trạng thái gần nhất
     - runtime launcher kiểm tra process thật sự còn sống không trước khi tin vào trạng thái DB

4. Thiết kế launcher/runtime cho managed browser.
   - Tạo một launcher riêng trong infrastructure, ví dụ `managed_browser_runtime.py`.
   - Khi cần fallback:
     - tìm 1 managed browser `is_active=True`
     - nếu `running` và CDP endpoint reachable thì attach luôn
     - nếu `stopped/failed` hoặc endpoint không reachable thì start lại browser process
   - Start browser bằng command rõ ràng, ví dụ Chromium/Chrome với:
     - `--remote-debugging-port=<port>`
     - `--user-data-dir=<dir>`
     - `--headless=new` nếu config bật headless
   - Sau khi start, runtime health-check endpoint `http://host:port/json/version` hoặc attach test qua Playwright.
   - Nếu attach thành công thì update `status=running`, `pid`, `last_started_at`, `last_seen_at`.

5. Refactor runtime chọn browser hiện tại.
   - Cập nhật [playwright_runtime.py](/home/mk/Documents/Github/Dogo-Corp/web-spider/src/infrastructure/adapters/spiders/helpers/playwright_runtime.py) để resolve theo thứ tự:
     - active external `cdp_connection`
     - active `managed_browser`
   - Loại bỏ hoặc giảm vai trò của nhánh `playwright.chromium.launch(...)` trong task process.
   - Mục tiêu là task chỉ `attach_over_cdp()` vào một browser đã tồn tại, không tự launch browser riêng.

6. Thêm màn hình quản lý browser trong `Operations`.
   - Tạo page mới, ví dụ `Managed Browsers`.
   - UI nên có:
     - danh sách browser configs
     - trạng thái hiện tại `running/stopped/failed`
     - host/port/headless/user data dir
     - nút `Start`, `Stop`, `Edit`, `Delete`
     - form tạo mới browser config
   - Có thể thêm note command/runtime summary tương tự `CDP Connections`, nhưng không yêu cầu user tự chạy lệnh vì app có thể start local browser thay họ.

7. Ràng buộc an toàn vận hành.
   - Không cho xóa config khi browser đang `running` trừ khi user stop trước.
   - Không cho start 2 config dùng cùng `port`.
   - Khi process chết ngoài ý muốn, UI hoặc runtime lần sau phải tự phát hiện và chuyển `status` về `failed` hoặc `stopped`.
   - Nên có cơ chế chọn duy nhất 1 browser active làm fallback mặc định, hoặc ít nhất runtime có rule ưu tiên rõ ràng theo `is_active`, `updated_at`, hoặc `id`.

8. Tích hợp với worker và quy trình crawl.
   - Product spider và các spider Playwright khác chỉ gọi runtime chung để lấy page.
   - Nếu external CDP không có, runtime attach vào managed browser local.
   - Nếu managed browser cũng không có config active, khi đó mới fail có kiểm soát với message rõ ràng, hoặc tùy chọn auto-create default managed browser config sau này.

9. Kế hoạch triển khai thực tế.
   - Bước 1: thêm model/migration `managed_browser`
   - Bước 2: thêm repository/service/facade
   - Bước 3: thêm launcher/runtime start/stop/check browser local
   - Bước 4: refactor `playwright_runtime.py` sang `external CDP -> managed browser CDP`
   - Bước 5: thêm UI page `Managed Browsers`
   - Bước 6: manual test fallback không có CDP external
   - Bước 7: lint + kiểm tra memory/process behavior

# Validation
- Khi không có `CDP Connection` active, một job crawl đầu tiên sẽ start managed browser local, các job sau attach reuse browser đó thay vì launch process browser mới mỗi task.
- Trong DB có record `managed_browser` lưu được config và trạng thái chạy gần nhất.
- UI `Managed Browsers` cho phép tạo config, start/stop browser và xem status hiện tại.
- `playwright_runtime.py` không còn phụ thuộc vào cache browser theo process để reuse local fallback.
- Số lượng browser process ngoài hệ thống ổn định khi queue crawl chạy dài, không tăng tuyến tính theo số task.
- `poetry run lint` thoát với code 0.
